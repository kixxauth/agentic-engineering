# Kixx Dev Assistant — HTML & CSS Guide
How to write HTML, structure pages, and work with styles in this project.

## 1. High-Level Architecture

Pages are served by the **Hyperview** plugin. The rendering pipeline works like this:

1. A route in `routes/main.jsonc` maps a URL pattern to request handlers.
2. The request handler prepares data (the "view model") and passes it to Hyperview.
3. Hyperview selects a **base template** from `templates/base-templates/` and injects the matching **page template** from `pages/` into the `{{unescape body}}` slot.
4. **Partials** in `templates/partials/` (like `html-header.html` and `styles.css`) are included with the `{{> partial-name }}` Handlebars syntax.

### Directory Map

```
templates/
  base-templates/
    base.html              <- Main app shell (sidebar + content grid)
    style-guide-base.html  <- Minimal shell for the style guide page
  partials/
    html-header.html       <- <head> content (meta, icon fonts, CSS)
    styles.css             <- Compiled Tailwind CSS output (do not edit directly)
    script.js              <- Shared browser-side JavaScript

pages/
  page.html                <- Home page content (matches "/" route)
  style-guide/
    page.html              <- Style guide page content
  contexts/
    id/
      page.html            <- Single context view (matches "/contexts/:id")

css/
  manifest.css             <- Tailwind 4 config + theme tokens
  custom.css               <- Custom component CSS (buttons, cards, fields, etc.)
```

## 2. Base Templates & Page Structure

### The Main Base Template (`base.html`)

Most pages use this template. It provides a two-column CSS Grid layout with a sidebar navigation on the left and a `<main>` content area on the right.

```html
<!doctype html>
<html lang="en-US">
  <head>
    {{> html-header.html }}
  </head>
  <body style="display:grid;grid-template-columns:auto 1fr;">

    <!-- Sidebar -->
    <div style="margin:1.5rem 0 0 1.5rem;min-width:15rem;">
      <div class="sidebar-nav space-y-4" style="position:sticky;top:1.5rem;">
        <!-- Navigation sections here -->
      </div>
    </div>

    <!-- Main Content -->
    <main style="display:grid;margin:1.5rem 1.5rem 0 1.5rem;">
      {{unescape body }}
    </main>

    <script type="module">
      {{> script.js }}
    </script>
  </body>
</html>
```

> **Key point:** Top-level page layout (the sidebar/content grid) uses inline `style` attributes on the `<body>` and its direct children. This is intentional — structural layout is expressed inline, while component styling uses CSS classes.

### Writing a Page Template

Page templates in `pages/` contain only the content that goes inside the `<main>` element. They should NOT include `<html>`, `<head>`, or `<body>` tags.

```html
<!-- pages/page.html — the home page -->
<div class="card card--elevated p-4">
    <h2 class="text-lg font-semibold text-primary mb-4">Create new context</h2>

    <form action="{{ newContextForm.action }}" method="{{ newContextForm.method }}" class="space-y-4">
        <!-- Form fields -->
        <div>
            <input type="submit" class="button button--primary" value="Create">
        </div>
    </form>
</div>
```

### Kixx Templating

Templates use the Kixx Handlebars syntax. Important patterns:

- `{{> partial-name.html }}` — include a partial from `templates/partials/`
- `{{unescape body }}` — render the page content (unescaped HTML)
- `{{ variable }}` — output an escaped variable
- `{{#each items as |item|}} ... {{/each}}` — iteration
- `{{#with object}} ... {{/with}}` — scope into an object
- `{{#if condition}} ... {{else}} ... {{/if}}` — conditionals
- `{{#ifEqual a "b"}} ... {{/ifEqual}}` — equality comparison (custom helper)

## 3. CSS System Overview

The project uses a three-layer CSS system:

| Layer | File | Purpose |
|-------|------|---------|
| Theme tokens | `css/manifest.css` | Material Design 3 color tokens defined via Tailwind's `@theme` directive |
| Custom components | `css/custom.css` | Hand-written CSS for buttons, cards, text fields, lists, sidebar nav, etc. |
| Utility classes | Generated by Tailwind | Standard Tailwind utility classes (spacing, typography, color, flex, grid) |

The dev server compiles `css/manifest.css` (which imports `custom.css` and scans `pages/` and `templates/`) into `templates/partials/styles.css`. This compiled output is inlined into the `<head>` of every page via the `html-header.html` partial.

> **Do NOT edit `templates/partials/styles.css` directly.** It is a generated file. Edit `css/manifest.css` or `css/custom.css` instead. If you edit `manifest.css`, you need to restart the dev server.

## 4. Material Design 3 Color System

All colors come from the [Material Design 3 color system](https://m3.material.io/styles/color/system/overview). Theme tokens are generated by the [Material Theme Builder](https://material-foundation.github.io/material-theme-builder/) and defined in `css/manifest.css` as CSS custom properties inside Tailwind's `@theme` block.

### Color Token Naming Convention

Every color comes in a pair: a *container/surface* color and an *on-* color for content placed on top of it. Always use the matching pair together.

| Background Class | Text Class | Use For |
|------------------|------------|---------|
| `bg-primary` | `text-on-primary` | Primary actions, key UI elements |
| `bg-primary-container` | `text-on-primary-container` | Less prominent primary areas |
| `bg-secondary` | `text-on-secondary` | Secondary actions |
| `bg-secondary-container` | `text-on-secondary-container` | Less prominent secondary areas |
| `bg-tertiary` | `text-on-tertiary` | Accents and contrast |
| `bg-tertiary-container` | `text-on-tertiary-container` | Less prominent accents |
| `bg-error` | `text-on-error` | Error states |
| `bg-error-container` | `text-on-error-container` | Less prominent error areas |
| `bg-surface` | `text-on-surface` | Default content areas |
| `bg-surface-container*` | `text-on-surface` | Layered surfaces (lowest to highest) |
| `bg-inverse-surface` | `text-inverse-on-surface` | High-contrast callouts |

### Surface Container Hierarchy

Surface containers provide visual depth without shadows. They go from darkest to lightest:

1. `bg-surface-container-lowest` — Deepest background
2. `bg-surface-container-low` — Used for elevated cards
3. `bg-surface-container` — Default container (sidebar nav)
4. `bg-surface-container-high` — Filled cards
5. `bg-surface-container-highest` — Highest prominence

### Text-Only Color Classes

For text on the default background, use these standalone text color classes:

- `text-on-surface` — Default body text
- `text-on-surface-variant` — Secondary/muted text
- `text-primary` — Headings, links, emphasis
- `text-secondary` — Navigation labels, supporting text
- `text-tertiary` — Accents
- `text-error` — Error states

## 5. Icons

Icons use [Google Material Symbols Rounded](https://fonts.google.com/icons?icon.set=Material+Symbols&icon.style=Rounded) loaded as a web font. The font is loaded in `templates/partials/html-header.html`.

### Basic usage

```html
<span class="icon text-xl text-on-surface-variant">search</span>
```

The `.icon` class sets the font family to `Material Symbols Rounded`. The text content is the icon name (lowercase, underscores for spaces). Size the icon with Tailwind text-size classes like `text-lg` or `text-xl`.

### Adding new icons

Available icons are limited to those listed in the `icon_names=` query parameter of the Google Fonts URL in `html-header.html`. To add a new icon:

1. Find the icon name at [fonts.google.com/icons](https://fonts.google.com/icons)
2. Add it to the `icon_names=` list in **alphabetical order**

Currently available icons: `add`, `arrow_right`, `cancel`, `check_circle`, `close`, `double_arrow`, `download`, `error`, `fast_forward`, `history`, `info`, `pause`, `play_arrow`, `search`, `stop`.

## 6. Component Reference

### 6.1 Buttons

Buttons follow [Material Design 3 button guidelines](https://m3.material.io/components/buttons/overview). Defined in `css/custom.css`.

#### Button Types

```html
<!-- Text button (default) -->
<button class="button">Text Button</button>

<!-- Outlined button -->
<button class="button button--outlined">Outlined Button</button>

<!-- Primary (filled) button -->
<button class="button button--primary">Primary Button</button>

<!-- Secondary button -->
<button class="button button--secondary">Secondary Button</button>
```

#### Button Sizes

```html
<!-- Default size (height: 2.5rem) -->
<button class="button button--primary">Default</button>

<!-- Small size (height: 2rem) -->
<button class="button button--primary button--small">Small</button>
```

#### Buttons with Icons

Use two `<span>` children. The first `<span>` becomes the icon (rendered in Material Symbols font); the second is the label.

```html
<button class="button button--primary">
    <span>add</span><span>Create</span>
</button>

<a href="#" class="button button--outlined">
    <span>history</span><span>View History</span>
</a>
```

#### Disabled State

```html
<button class="button button--primary" disabled>Disabled</button>
```

#### On Different Elements

The `.button` class works on `<button>`, `<a>`, and `<input type="submit">`:

```html
<button class="button button--primary">Button</button>
<a href="#" class="button button--secondary">Link Button</a>
<input type="submit" class="button button--outlined" value="Submit">
```

> **Note:** `<input type="submit">` elements cannot contain icons because they are void (self-closing) elements.

### 6.2 Cards

Cards follow [Material Design 3 card guidelines](https://m3.material.io/components/cards/overview). Defined in `css/custom.css`.

#### Card Variants

```html
<!-- Outlined card -->
<div class="card card--outlined">
    <div class="card__inner">
        <p>Content here</p>
    </div>
</div>

<!-- Elevated card (with drop shadow) -->
<div class="card card--elevated">
    <div class="card__inner">
        <p>Content here</p>
    </div>
</div>

<!-- Filled card -->
<div class="card card--filled">
    <div class="card__inner">
        <p>Content here</p>
    </div>
</div>

<!-- Error card -->
<div class="card card--error">
    <div class="card__inner">
        <p>Error message</p>
    </div>
</div>
```

#### Full-Bleed Sections

Cards clip overflow (`overflow: hidden`), so direct children that are NOT wrapped in `.card__inner` extend edge-to-edge. Use this for colored header areas:

```html
<div class="card card--outlined">
    <div class="bg-primary-container">
        <p class="p-4 text-sm text-on-primary-container">Full Bleed Header</p>
    </div>
    <div class="card__inner">
        <h3 class="text-xl font-bold">Card Title</h3>
        <p>Card content</p>
    </div>
</div>
```

> **Tip:** Cards can also be styled with Tailwind utility classes directly. For example, `<div class="card card--elevated p-4">` applies uniform padding instead of using `.card__inner`.

### 6.3 Lists

Lists follow [Material Design 3 list guidelines](https://m3.material.io/components/lists/overview). Defined in `css/custom.css`.

#### List Sizes

```html
<ul class="bg-surface-container py-2 rounded-sm">
    <!-- Single-line list item (min-height: 4rem) -->
    <li class="list-item">
        <span class="list-item__center text-lg">Item Name</span>
    </li>

    <!-- Two-line list item (min-height: 5.6rem) -->
    <li class="list-item list-item--lg">
        <div class="list-item__center">
            <h3 class="text-lg">Title</h3>
            <p class="text-sm text-on-surface-variant">Supporting text</p>
        </div>
    </li>

    <!-- Three-line list item (top-aligned, min-height: 5.6rem) -->
    <li class="list-item list-item--xl">
        <div class="list-item__center">
            <h3 class="text-lg">Title</h3>
            <p class="text-sm text-on-surface-variant font-semibold">Subtitle</p>
            <p class="text-sm text-on-surface-variant mt-2">Description text...</p>
        </div>
    </li>
</ul>
```

#### List Item Anatomy

List items use flexbox. The `.list-item__center` class applies `flex-grow: 1` to fill available space. Optional leading/trailing elements go before/after it:

```html
<li class="list-item">
    <!-- Leading: icon or image -->
    <span class="icon text-xl text-on-surface-variant">add</span>

    <!-- Center: main content (grows to fill space) -->
    <span class="list-item__center text-lg">Item Name</span>

    <!-- Trailing: meta text or icon -->
    <span class="text-xs text-on-surface-variant">2,857</span>
</li>
```

For images as the leading element:

```html
<li class="list-item list-item--xl">
    <div class="w-32 shrink-0"><img src="..." alt="..."></div>
    <div class="list-item__center">
        <h3 class="text-lg">Title</h3>
        <p class="text-sm text-on-surface-variant">Description</p>
    </div>
    <span class="icon text-xl text-on-surface-variant">arrow_right</span>
</li>
```

### 6.4 Text Fields

Text fields are defined in `css/custom.css`.

#### Basic Text Input

```html
<div class="text-field max-w-sm">
    <label>Field Label</label>
    <input type="text" name="fieldName" placeholder="Placeholder">
</div>
<span class="text-field-supporting">Optional helper text</span>
```

#### Text Input with Icons

Wrap the input in a `.text-field__container` and add `<span>` elements for the icon(s). The icon content is a Material Symbol name:

```html
<!-- Left icon -->
<div class="text-field max-w-sm">
    <label>Search</label>
    <div class="text-field__container">
        <span>search</span>
        <input type="text" name="search" placeholder="Search...">
    </div>
</div>

<!-- Right icon -->
<div class="text-field max-w-sm">
    <label>Search</label>
    <div class="text-field__container">
        <input type="text" name="search" placeholder="Search...">
        <span>cancel</span>
    </div>
</div>

<!-- Both icons -->
<div class="text-field max-w-sm">
    <label>Search</label>
    <div class="text-field__container">
        <span>search</span>
        <input type="text" name="search" placeholder="Search...">
        <span>cancel</span>
    </div>
</div>
```

#### Textarea

```html
<div class="text-field max-w-sm">
    <label>Message</label>
    <textarea name="message" placeholder="Type here..."></textarea>
</div>
```

#### Error State

Add `text-field--error` to the `.text-field` wrapper. Use `text-field-supporting--error` for error messages:

```html
<div class="text-field text-field--error max-w-sm">
    <label>Email</label>
    <div class="text-field__container">
        <span>search</span>
        <input type="text" name="email" placeholder="you@example.com">
        <span>cancel</span>
    </div>
</div>
<p class="text-field-supporting text-field-supporting--error inline-flex items-center gap-1">
    <span class="icon text-lg">error</span><span>Please enter a valid email</span>
</p>
```

#### Disabled State

```html
<div class="text-field max-w-sm">
    <label>Disabled Field</label>
    <input type="text" name="text" placeholder="..." disabled>
</div>
```

### 6.5 Sidebar Navigation

Used in the main app shell and the style guide. Defined in `css/custom.css`.

```html
<nav class="sidebar-nav">
    <span class="sidebar-nav__section-header text-sm uppercase text-on-surface-variant">
        Section Title
    </span>
    <ul class="sidebar-nav__list">
        <li><a href="/path">Link Label</a></li>
        <li><a href="/other">Another Link</a></li>
    </ul>
</nav>
```

When a `.sidebar-nav__section-header` precedes the list, link items are automatically indented with extra left padding.

### 6.6 Code Blocks

```html
<div class="card card--outlined code-block">
    <pre><code>your code here</code></pre>
</div>
```

### 6.7 Text Content Areas

Wrap blocks of prose text in a `.text-content` container to get automatic link underlines and code sizing:

```html
<div class="text-content">
    <p>Links like <a href="#">this one</a> are automatically underlined.</p>
    <p>Inline <code>code</code> is sized at 0.9em.</p>
</div>
```

## 8. Custom CSS vs. Tailwind: When to Use Which

| Use Case | Approach |
|----------|----------|
| Reusable component (button, card, list, form field) | Custom CSS class in `css/custom.css` with BEM-like naming (`.block`, `.block--modifier`, `.block__element`) |
| One-off spacing, sizing, color | Tailwind utility classes |
| Top-level page layout (grid, positioning) | Inline `style` attributes |
| Theme color tokens | Defined in `css/manifest.css` `@theme` block; consumed via Tailwind classes like `bg-primary` or `text-on-surface` |

### Custom CSS Naming Conventions

The project uses a BEM-like convention:

- `.component` — Block (e.g., `.button`, `.card`, `.text-field`)
- `.component--variant` — Modifier (e.g., `.button--primary`, `.card--elevated`)
- `.component__element` — Child element (e.g., `.card__inner`, `.list-item__center`)

## 9. Common Page Patterns

### Section with Header

```html
<section>
    <header class="text-content">
        <h2 class="text-3xl text-primary">Section Title</h2>
        <p class="mt-2">Description with <a href="#">a link</a>.</p>
    </header>

    <div class="mt-6">
        <!-- Section content -->
    </div>
</section>
```

### Two-Column Layout (Content + Code Example)

Used throughout the style guide. A responsive grid with content on the left and a code block on the right:

```html
<div class="mt-6 gap-4 grid grid-cols-1 2xl:grid-cols-2">
    <div class="card bg-surface">
        <div class="card__inner">
            <!-- Visual content -->
        </div>
    </div>
    <div class="card card--outlined code-block">
        <pre><code>Code example</code></pre>
    </div>
</div>
```

### Form Pattern

Forms use a card container with vertical spacing between fields:

```html
<div class="card card--elevated p-4">
    <h2 class="text-lg font-semibold text-primary mb-4">Form Title</h2>

    <form action="/endpoint" method="POST" class="space-y-4">
        <!-- Text field -->
        <div>
            <div class="text-field">
                <label>Field Label</label>
                <input type="text" name="fieldName">
            </div>
        </div>

        <!-- Text field with error (conditionally rendered) -->
        <div>
            <div class="text-field text-field--error">
                <label>Field Label</label>
                <input type="text" name="fieldName">
            </div>
            <p class="text-field-supporting text-field-supporting--error flex items-center gap-1">
                <span class="icon text-lg">error</span><span>Error message</span>
            </p>
        </div>

        <!-- Submit button -->
        <div>
            <input type="submit" class="button button--primary" value="Submit">
        </div>
    </form>
</div>
```

### Page Layout with Sidebar

The style guide demonstrates a self-contained sidebar + content layout using CSS Grid:

```html
<div style="display:grid;grid-template-columns:auto 1fr;">
    <!-- Sidebar -->
    <div style="padding:0 1.5rem;">
        <nav class="sidebar-nav" style="position:sticky;top:1.5rem;min-width:15rem">
            <ul class="sidebar-nav__list">
                <li><a href="#section1">Section 1</a></li>
                <li><a href="#section2">Section 2</a></li>
            </ul>
        </nav>
    </div>

    <!-- Content -->
    <main class="pt-6 pr-6 pl-4 pb-6 space-y-12">
        <section>...</section>
        <section>...</section>
    </main>
</div>
```

## 10. Rendered Content (Assistant Turns)

When rendering Markdown-to-HTML content (such as Claude assistant responses), wrap it in `.assistant-turn__text` to get proper heading sizes, paragraph spacing, and list styles:

```html
<div class="assistant-turn__text">
    <!-- Rendered Markdown HTML goes here -->
    <h2>Response Heading</h2>
    <p>Response paragraph.</p>
    <ul>
        <li>Bullet point</li>
    </ul>
    <pre><code>code block</code></pre>
</div>
```

This class restores browser-default-like typography (margins, list bullets, heading sizes) that Tailwind's reset strips out.

## 11. Site Width & Responsive Layout

The site has a fixed max width defined in `css/custom.css`:

- `--site-width: 760px` — maximum content width
- `--layout-right-left-margin: 2vw` — side margins on small screens

The `.site-width-container` class centers content at the site width on screens wider than 760px, and uses 2vw side margins on smaller screens.

```html
<div class="site-width-container">
    <!-- Content is centered at max 760px -->
</div>
```

The `2xl:` Tailwind breakpoint (96rem / 1536px) is used for wide-screen two-column grids: `grid-cols-1 2xl:grid-cols-2`.

## 12. Quick Reference Checklist

- Page templates go in `pages/` — no `<html>`/`<head>`/`<body>` tags
- Always pair background + text colors (`bg-primary` + `text-on-primary`)
- Use custom CSS classes for reusable components; Tailwind utilities for one-off styling
- Use inline `style` attributes for top-level structural layout (grids, sticky positioning)
- Add new custom component CSS to `css/custom.css`
- Add new Tailwind theme tokens to the `@theme` block in `css/manifest.css` (requires dev server restart)
- Never edit `templates/partials/styles.css` directly
- New icons must be added to the `icon_names=` list in `html-header.html` (alphabetical order)
- Browser-side JavaScript must be ES2015 (no async/await)
- Refer to the live style guide page at `/style-guide` for visual examples of all components
